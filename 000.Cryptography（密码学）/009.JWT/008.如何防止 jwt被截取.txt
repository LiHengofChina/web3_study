如果 JWT 被其他人截取到，且没有额外的保护机制，
理论上是可以 "伪造登录" 或 "冒充合法用户" 的。

//=============================================== 通常会采取以下措施来提高安全性：
//=============================================== 通常会采取以下措施来提高安全性：

（1）必须使用 HTTPS 来确保 JWT 在传输过程中不会被中间人截获。


（2）设置较短的过期时间 (exp)


（3）在服务器端实现 Token 黑名单机制
	可以在服务器端实现一个 Token 黑名单机制，
	将已经被滥用或登出的 JWT 加入黑名单，避免被再次使用。

（4）使用刷新 Token（Refresh Token）

（5）签名密钥的安全性


（6）绑定 Token 与客户端信息

（7） HttpOnly 和 Secure 标记
	如果没有保护机制，JWT 被截取后确实可能被滥用，
	但通过使用 HTTPS、短期有效期、Token 黑名单、刷新 Token 和 绑定客户端信息 等措施，
	可以显著提高 JWT 的安全性，防止被攻击者利用。
