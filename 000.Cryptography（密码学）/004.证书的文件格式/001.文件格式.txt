而其他诸如 .cer、.crt、.pfx、.p12、.jks .asc 等都是存储密钥、证书等内容的文件格式。
这些文件格式通常会使用 PEM 或 DER 作为编码方式来存储数据。

//========================================= 文件格式：
//========================================= 文件格式：

.pem						：  通常用于存储证书或密钥，使用 PEM 编码。

.jks（Java Keystore）		：  Java 密钥库文件，内容通常是 DER 编码，但属于 Java 专有格式。
.cer / .crt					：	存储公钥证书，内容可以是 PEM 或 DER 编码。
.csr						：	证书签名请求，通常是 PEM 编码。

p12 / .pfx（PKCS12 格式）	：  用于打包私钥、公钥证书和证书链，内容通常是 DER 编码并加密。



.kbx						:   PGP 的密钥库类似于 Java 中的 .jks（Java KeyStore）文件，
								它是 GnuPG（GPG）中用于存储公钥的一种 密钥库 文件。
								它保存了你导入或生成的所有公钥信息。
								虽然它与 .jks 的具体用途和格式不同，但功能上有相似之处，都用于存储加密密钥。
								示例：/home/liheng/.gnupg/pubring.kbx

//=========================================
//=========================================
.asc（ASCII Armored）		：  PGP 格式的公钥、私钥或加密文件，使用 ASCII Armored 编码（PEM 的一种变体）。
								它通常包含公钥、私钥或加密消息的数据，并以文本形式存储。
								用于 PGP（Pretty Good Privacy）和 GPG（GNU Privacy Guard）密钥文件，
								文件头通常以 "-----BEGIN PGP PUBLIC KEY BLOCK-----" 开始。

.pgp 						.pgp 文件通常是二进制格式，用于存储 PGP 加密的消息或密钥。
								它们可以包含相同的信息，但以二进制形式存储。
								//.asc 和 .pgp 的区别确实类似于 .pem 和 .der 的区别。
