//======================================================= JCE 限制（Java 8 及更低版本）
在 Java 8 及更早版本，默认情况下，Java Cryptography Extension (JCE) 对加密算法的密钥长度有限制，
AES-256 密钥长度不被默认支持。
你需要安装 Java Cryptography Extension (JCE) 无限制强度策略文件，以解除密钥长度的限制。


如果你使用 AES-256 进行加密，而后端使用的 JDK 版本没有安装 JCE 无限制策略文件，解密就会失败。


//======================================================= 解决方法：
//======================================================= 解决方法：

检查后端 JDK 版本是否低于 Java 9。如果是 Java 8 或更早版本，确保安装了 JCE 无限制强度策略文件。

在 Java 8 的环境下，你可以通过安装这些策略文件来解除密钥长度限制：
下载并安装 JCE 无限制策略文件。
将下载的 US_export_policy.jar 和 local_policy.jar 文件替换到 JDK 的 jre/lib/security 目录中。

默认支持：AES-128 不会遇到 JCE 无限制策略文件的密钥长度限制问题，Java 默认支持 128 位密钥的 AES 加密和解密。
性能和安全性平衡：AES-128 提供足够的安全性，同时性能也优于 AES-256，尤其是在处理速度上有优势。
兼容性：AES-128 被广泛支持，适用于大多数系统和环境，不会遇到兼容性问题。
