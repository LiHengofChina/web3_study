（1）在 JSON Web Token (JWT) 中，HS256 和 HS512 常用于对 JWT 的头部和负载进行签名。
	 签名是为了确保 JWT 在传输过程中没有被篡改。

（2）使用时，签名的格式如下：
	 HMAC-SHA256(header + payload, secret_key)
	 通过 HS256 或 HS512 生成的签名可以在服务器端进行验证，确保 JWT 的完整性。
