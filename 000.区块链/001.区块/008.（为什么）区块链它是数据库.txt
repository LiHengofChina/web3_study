

关系型数据库存放的数据是行、列的形式称为表，而
区块数据库它是文件形式吗

//============================================== 传统关系型数据库（如 MySQL、PostgreSQL）：
//============================================== 传统关系型数据库（如 MySQL、PostgreSQL）：

数据结构：表结构

数据表现：行和列（类似 Excel 表格）

//==============================================
//==============================================

| id | name  | balance |
|----|-------|---------|
| 1  | Alice | 100.0   |
| 2  | Bob   | 50.0    |

使用 SQL 语言操作
支持强查询、复杂筛选、高度结构化数据

//============================================== 区块链中的“区块数据”：
//============================================== 区块链中的“区块数据”：
不是表格形式，而是以
	""
	 '文件'或'区块结构'
	""存储

每个区块像是一个“JSON 文件”或“自定义数据结构”
区块头（metadata）
交易列表（transactions）
{
  "previousHash": "...",
  "timestamp": "...",
  "transactions": [
    {"from": "Alice", "to": "Bob", "amount": 1.2},
    {"from": "Charlie", "to": "Eve", "amount": 0.8}
  ]
}
//============================================== 总结一句话：
//============================================== 总结一句话：

关系型数据库：结构化表格数据（行列）
区块链数据：结构化文件或对象数据（像 JSON 区块）
所以区块链不是用传统“表”的方式存储，而是一个个区块文件串起来形成链。
