每条公链最初创建时都会定义一个 “主币”（native coin），用于：
	支付手续费
	奖励矿工/节点
	治理投票
	/**
	举例：
		比特币链上的主币是：BTC
		以太坊链上的主币是：ETH
		BNB 链上的主币是：BNB
	*/
//============ 代币：在主链上发行的 "应用币"
//============ 代币：在主链上发行的 "应用币"


区块链是开放平台，任何人都可以在上面 "部署智能合约"，创建 "自己的代币（Token）"

所以会出现很多：
	游戏币、平台币、项目币、稳定币、诈骗币…

举例：
	在以太坊上，你可以看到成千上万个代币：USDT、UNI、MKR、SHIB、PEPE……
	这些是通过ERC-20 标准合约发行出来的

//============ 为什么这么多？
//============ 为什么这么多？

原因很简单：
	区块链是 去中心化平台，任何人都能 "在链上发币"
		只要部署合约，就可以创建属于自己的 "虚拟货币"
		这就像互联网允许你自己建网站一样：只要你想，就能搞一个


"每种链" 上通常有主流的虚拟货币，
	但 "也可以支持其他代币" 或 "资产"。

例如，以太坊链上有以太坊（ETH），但也支持许多基于它的代币（如USDT、DAI）。

一、每个区块确实只有一个 nonce（用于挖矿）
	是挖出这个区块、获得写入权限的证明。
	它对应的是 "系统主币的奖励"（比如 BTC 奖励 6.25 个）
	所以：
		nonce 是为了达成 "共识"
		是 "挖出一个新区块" 的凭证（对谁写入的奖励）

二、代币并不需要挖矿，它们是 "账户数据" 保存在链上的

	比如：以太坊链上你可以发一个自己的代币 MyToken，然后用 "合约代码" 管理 "这些币的余额转账"。
	这些 "代币资产" 是保存在 "某个智能合约账户" 里的数据状态中，比如：
	"用户 A" 的地址：1000 个 MyToken
	"用户 B" 的地址：200 个 MyToken
	它们只是 "数据而已"，不是挖出来的，也没有 nonce。

	//"代币"也是 "加密货币"，只是它依附在某条主链上，不是自己独立挖矿产生的。

//=========================================== 代币（Token）的几个关键特点：
//=========================================== 代币（Token）的几个关键特点：

（1）不用挖矿产生
	  它们通常是由智能合约一次性或分批发行出来的（比如 ERC-20 标准）。
（2）供应量可控 / 可通胀
	  发行方可以增发或销毁代币，是否通胀取决于项目的“通证经济模型”。

（3）运行在主链上
	运行在主链上
	比如：
		USDT（泰达币）是运行在以太坊、Tron、Solana 等链上的代币。
		它们依赖主链的安全性和共识机制。
		/**
			USDT 在以太坊、Tron、Solana 等链上存在多个版本，技术上不是同一个 USDT，
				它们代表的是同一种资产（锚定美元 1:1）；
				由同一个机构 Tether 发行与管理；
				可以通过官方或交易所进行链间兑换（例如把 Tron-USDT 换成 ETH-USDT）。
		*/
		
用途灵活
	常用于：
		支付手段（如 USDT、DAI）
		治理投票（如 UNI、AAVE）
		奖励激励（如某些 DeFi 项目代币）
		NFT 交易媒介（如一些游戏币）


//=========================================== 主链币（主币 / 原生币）
//=========================================== 主链币（主币 / 原生币）
依赖“区块”和“链”本身存在。
是区块链系统运行和激励的基础货币。

例如：
	BTC → 比特币主链的原生币。
	ETH → 以太坊主链的原生币。
用于支付交易手续费（如 ETH 支付 gas），奖励矿工，维护网络等。

//=========================================== 代币（Token）
//=========================================== 代币（Token）

通过智能合约发行，存在于已有的区块链之上（如以太坊、BSC）。
依附于主链的运行机制，不拥有自己的独立区块链。

举例：
	USDT-ERC20（在以太坊上）
	SHIB、UNI、MATIC（都是代币）
需要主币（如 ETH）支付手续费（因为智能合约运行在主链上）。

//=========================================== 代币记录的位置
//=========================================== 代币记录的位置

代币仍然记录在区块中，只不过：
	它不是**“原生记录”，而是由智能合约驱动的记录**；
	代币的交易、发行、转账等信息，也会被打包进所在区块链的区块中；
	所以它依然“上链”且“写入区块”，只是不是挖出来的、也不影响区块结构。

代币仍然记录在区块里，
只是通过智能合约操作写入，而不是作为主币直接挖出来。
也就是说：代币和区块也有关联，只是方式不同。

//===========================================
//===========================================
代币的交易记录也记录在区域的 transactions 里面，只是它先挖到主币，然后进行的写入

代币的交易记录也写进区块链的 transactions 中，只不过这些交易是智能合约驱动的。

//=========================================== 具体流程如下：
//=========================================== 具体流程如下：

（1）矿工/验证者挖出新区块（或打包区块）；
（2）他们从交易池中挑选交易，包括：
	普通的主币交易（如 ETH 转账）；
	智能合约交易（比如 USDT 转账、DEX 交易等）；
（3）代币交易也是交易的一种，只不过它的 to 地址是智能合约地址（不是普通用户地址）；
（4）这些交易一并被打包进区块的 transactions 数组中；
（5）当区块被写入链中，这些代币交易就永久记录在链上了。

//=========================================== 举个例子（以太坊 + USDT）：
//=========================================== 举个例子（以太坊 + USDT）：

	发起 USDT 转账（ERC20 代币）其实是调用智能合约函数 → 也是一条交易；
	它也会被打包进某个区块；
	浏览器上你也能看到这些代币交易记录在区块里，只是显示格式不同（比如 Token Transfers）。

//=========================================== 总结：
//=========================================== 总结：
	所有交易（包括主币、代币）都会进入区块的 transactions 中，
	只是代币是通过合约控制的交易，本质仍然是写进区块链的数据。

"先挖主币，再写入代币交易" 的顺序，本质上也是对的——
矿工打包的是所有交易，获得的是主币奖励，而代币只是顺带记录的内容。
